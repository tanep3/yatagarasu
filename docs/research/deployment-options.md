# デプロイ方式調査メモ（初版）

- 作成日: 2026-02-20
- 対象: Tapo連携ロボットの常駐実行

## 調査結論

- 現段階の最適解は「ハイブリッド構成」。
- 理由は、デバイス制御の低遅延要件と、外部依存コンポーネントの再現性要件を同時に満たせるため。

## 判断根拠

1. 音声入力とトリガ監視は、ホスト実行の方がデバッグしやすい。
2. VoiceVox/go2rtc はコンテナ化で依存管理が安定する。
3. すべてDocker化するとデバイス境界の調整コストが先に増える。

## 次の検証項目

1. go2rtcの双方向音声で対象モデル別の接続差分を確認する。
2. systemd unit再起動設定で、RTSP断からの復帰時間を測定する。
3. 音声応答の体感遅延（先頭発話まで）を定量測定する。
